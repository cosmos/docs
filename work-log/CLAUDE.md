# Work Log Instructions

Each branch gets its own `.md` file in this directory, named after the branch (e.g. `consolidation-final.md`).

## How to make entries

- Log after completing each meaningful task, not after every individual file edit
- Keep entries short — what changed and why, not how
- Use bullet points
- Include the date (format: `YYYY-MM-DD`)
- Group related changes under one bullet if they were done together (e.g. "deleted X, fixed backlinks")

## Format

```md
## YYYY-MM-DD

- Brief description of change and reason
- Another change
```

## What to log

- Pages added, deleted, or moved
- Significant content changes (e.g. updated stale API references)
- Structural changes to docs.json navigation
- Backlink fixes that resulted from deletions/moves

## What not to log

- Typo fixes
- Minor wording edits
- Individual backlink repairs (mention them as part of the deletion/move entry)
