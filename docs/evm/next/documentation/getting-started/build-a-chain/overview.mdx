## What You'll Learn

This guide walks through:

1. **Pre-Genesis Configuration** - Set up chain identity, token economics, and module selection before initialization
2. **Genesis Configuration** - Configure all module parameters, enable precompiles, and set up the validator set
3. **Runtime Configuration** - Configure JSON-RPC, EVM settings, and node-specific parameters
4. **Network Launch** - Coordinate validator launch and monitor network health
5. **Post-Launch Operations** - Manage upgrades and parameter changes via governance

## Key Decisions

When building your chain, you'll make these critical architectural decisions:

**Token Economics**: Choose between 18-decimal tokens (direct EVM compatibility) or 6-decimal tokens (standard Cosmos pattern). This affects module selection and conversion logic.

**Precompile Selection**: Enable Cosmos functionality accessible from Solidity contracts - staking, governance, IBC, and more.

**Access Control**: Configure permissionless, permissioned, or restricted smart contract deployment.

**Validator Architecture**: Set up dedicated RPC nodes separate from validators for security and performance.

## Module Selection

<Note>
**18-Decimal Chains (Default)**: Uses standard Cosmos SDK `x/bank` module by default - no changes needed.

**6-Decimal Chains**: Must add `x/precisebank` module to handle fractional amounts during EVM conversions.
</Note>

## Validator Security

<Warning>
Validator nodes should NEVER expose RPC/API ports publicly. Only dedicated RPC nodes should serve public traffic.
</Warning>

See the [validator security guide](/docs/evm/next/documentation/getting-started/node-configuration#validator-nodes) for details.
